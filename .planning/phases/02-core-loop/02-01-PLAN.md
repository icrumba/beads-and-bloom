  <files>

  <action>
    1. Create `src/lib/event-bus.ts` -- a simple in-process pub/sub for SSE:
       - Use an EventEmitter pattern (Node.js events module)
       - Export functions: emitTaskEvent(event: TaskEvent), onTaskEvent(callback), offTaskEvent(callback)
       - TaskEvent type: { type: 'task:created' | 'task:updated' | 'task:deleted' | 'task:status' | 'task:progress', task: Task, timestamp: string }
       - This is the bridge between API mutations and SSE stream

    2. Create `src/app/api/tasks/route.ts`:
       - GET: Return all tasks from SQLite, ordered by columnOrder. Accept optional ?status= query param to filter by column. Return JSON array.
       - POST: Accept TaskCreateInput body. Generate UUID (crypto.randomUUID()), set status to 'backlog', columnOrder to max+1 in backlog, timestamps to now. Insert into SQLite. Emit 'task:created' event. Return 201 with the task.

    3. Create `src/app/api/tasks/[id]/route.ts`:
       - GET: Return single task by id. 404 if not found.
       - PATCH: Accept TaskUpdateInput body. Update only provided fields plus updatedAt. Emit 'task:updated' event. Return updated task. Handle columnOrder updates for drag-drop repositioning.
       - DELETE: Delete task by id. Also delete any child tasks (where parentId = id). Emit 'task:deleted' event. Return 204.

    4. Create `src/app/api/tasks/[id]/status/route.ts`:
       - PATCH: Accept { status: string, activityLabel?: string, costUsd?: number, tokensUsed?: number, errorMessage?: string }
       - This is the endpoint Claude sessions call to update their own status (agent-first API)
       - When status changes to 'running': set startedAt if null
       - When status changes to 'review' or 'done': set completedAt
       - When status changes to 'review' with errorMessage: this is an errored task
       - Emit 'task:status' event (distinct from task:updated so SSE clients can handle status changes specifically)
       - Return updated task

    5. Create `src/app/api/events/route.ts`:
       - GET: Return a ReadableStream with SSE format (text/event-stream content type)
       - On connect: send a 'connected' event with current timestamp
       - Subscribe to event bus, forward all TaskEvents as SSE data lines (JSON-encoded)
       - Use proper SSE format: `event: {type}\ndata: {json}\n\n`
       - Handle client disconnect: unsubscribe from event bus
       - Set headers: Cache-Control no-cache, Connection keep-alive

    6. All endpoints: use proper HTTP status codes, validate request bodies (return 400 for malformed input), handle errors with try/catch returning 500.
  </action>
  <verify>

    - `curl localhost:3000/api/events` streams SSE events (stays open, receives events when tasks are created/updated via another curl)
    - `curl -X PATCH localhost:3000/api/tasks/{id}/status -H 'Content-Type: application/json' -d '{"status":"queued"}'` returns updated task
