> Visual and interaction contract for the Storefront & Brand phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (base-nova style) |
| Preset | base-nova, neutral base, CSS variables enabled |
| Component library | radix-ui (via shadcn) |
| Icon library | lucide-react |

### Font Change Note

Layout.tsx currently loads Geist Sans. Phase 02 must replace it with Plus Jakarta Sans via `next/font/google`. Plus Jakarta Sans is a geometric sans-serif with softer curves that better fits the warm, charming brand voice. Load weights 400 and 600 only.

## Color

The shadcn neutral base (currently all achromatic oklch values) must be overridden with the brand palette. Only light mode is needed for the storefront -- dark mode tokens can remain as defaults but will not be used in Phase 02.

| Role | Value | OKLCH | Usage |
|------|-------|-------|-------|
| Dominant (60%) | #FAFAF8 | oklch(0.98 0.002 100) | Page background, product page background |
| Secondary (30%) | #F0EBE3 | oklch(0.94 0.012 80) | Card backgrounds, navigation background, footer, shipping/policy page panels |
| Accent (10%) | #3B9B8F | oklch(0.63 0.09 175) | See reserved-for list below |
| Accent foreground | #FFFFFF | oklch(1 0 0) | Text on accent-colored elements |
| Destructive | #DC2626 | oklch(0.577 0.245 27.325) | Keep shadcn default (not used in Phase 02 -- no destructive actions) |
| Foreground (text) | #1A1A1A | oklch(0.145 0 0) | Body text, headings |
| Muted foreground | #6B6B6B | oklch(0.48 0 0) | Secondary text: materials, care info, "Made to order" badge text |

### Accent Reserved For

The teal/ocean blue accent (#3B9B8F) is reserved exclusively for:

1. Primary CTA buttons ("Add to Cart" in future phases, but styled now for consistency)
2. Category tab active/selected state underline
3. Charity counter highlight number
4. "View Details" links on product cards (text color, not background)
5. Mobile menu active item indicator

Accent is NOT used for: card borders, section backgrounds, decorative elements, badges, or non-interactive text.

## Layout Specifications

### Mobile (< 768px) -- Primary viewport

- Product grid: 2 columns, 8px gap (D-01)
- Featured product: full width, larger card with bigger photo
- Page padding: 16px horizontal
- Header: logo + hamburger icon, 56px height, not sticky
- Product detail: photo carousel stacked on top, info below (D-04)
- About page: single column, alternating full-width photos and text blocks

### Tablet (768px - 1024px)

- Product grid: 3 columns, 16px gap
- Product detail: side-by-side layout -- photos 50% left, info 50% right (D-04)
- Page padding: 24px horizontal

### Desktop (> 1024px)

- Product grid: 4 columns, 16px gap, max-width 1200px centered
- Product detail: side-by-side, photos 55% left, info 45% right
- Page padding: 32px horizontal, max-width 1200px centered

## Interaction Patterns

### Photo Carousel (Product Detail)

- Horizontal swipe on mobile (CSS scroll-snap, snap mandatory)
- Dot indicators below: active dot is accent color, inactive dots are muted
- No autoplay -- user-initiated swipe only
- Minimum swipe distance: 50px to trigger snap

### Category Tabs

- Horizontal scrollable on mobile if tabs overflow
- Active tab: accent-colored underline (2px), semibold text
- Inactive tab: muted foreground text, regular weight
- Tab change fetches filtered products (server component re-render or client filter)

### Charity Counter Animation

- Count-up animation on page load, from 0 to actual total
- Duration: 2 seconds, ease-out curve
- Use CSS counter animation or a lightweight count-up approach (no heavy library)
- Format: "$[number] donated and counting!"
- Trigger: IntersectionObserver -- animate only when scrolled into view

### Mobile Navigation

- Hamburger icon triggers shadcn Sheet from right side
- Menu items: Shop, About, Shipping, Contact
- Sheet overlay dims the background
- Close on item tap or X button

### Product Card Tap

- Entire card is tappable (anchor wrapper)
- Subtle hover state on desktop: card lifts 2px with box-shadow transition (200ms ease)
- No hover effect on mobile (touch devices)
